Configure your bookmarklet repository

Repository Structure

Your repository should contain JavaScript files with the .js extension. Each file should follow the bookmarklet format:

javascript:
/* @title: My Bookmarklet */
/* @description: What it does */
/* @domains: example.com, *jira* */
(function() {
  // Your code here
})();

Metadata Tags

  • @title - Display name in the extension (optional, defaults to filename)
  • @description - Brief description shown below the title (optional)
  • @domains - Comma-separated list of domains where this bookmarklet should appear (optional, shows everywhere if omitted)
    • Exact match: github.com
    • Wildcard patterns: *jira*, *confluence*

README Documentation

Add detailed documentation by creating a README file next to your bookmarklet:

Auto-Grouping

Bookmarklets with common prefixes are automatically grouped:

  • utils_copy_title.js
  • utils_copy_url.js
  • utils_scroll_to_top.js

These will appear in a collapsible "utils" group when there are 3 or more with the same prefix.
